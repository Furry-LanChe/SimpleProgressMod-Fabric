# SimpleProgress  

一个轻量级的 Minecraft 自定义进度管理模组  

## 📖 模组介绍  

SimpleProgress 是一个基于 GUI 的进度管理模组，允许玩家在 Minecraft 中创建和管理自定义进度目标。支持多种进度类型、子进度系统和实时 HUD 显示，让你的游戏目标追踪变得简单高效。  

## ✨ 主要特性  

### 🎯 多种进度类型  
- **击杀**：追踪特定生物的击杀数量  
- **获得**：监控物品收集进度  
- **探索**：记录维度访问  
- **建筑**：统计方块放置数量  
- **附魔**：追踪附魔获取情况  

### 🌳 高级功能  
- 层次化子进度系统  
- 树状图统计可视化  
- 父子进度关系管理  

### 🎨 个性化界面  
- 6 种主题选择（默认/暗色/亮色/绿色/蓝色/紫色）  
- 可调整的 GUI 布局  
- 支持滚动的列表和鼠标交互  

### 📊 实时追踪  
- HUD 显示活跃进度  
- 右上角进度条显示  
- 完成时聊天栏通知  

### 💾 数据持久化  
- 自动保存到世界文件夹  
- 游戏重启后进度不丢失  
- 每个玩家独立数据  

## 🚀 使用方法  

### 打开界面  
- 使用 `/progress` 命令  
- 或通过模组配置界面访问  

### 创建进度  
1. 选择进度类型  
2. 输入标题和目标 ID  
3. 设置目标数量  
4. 点击"添加进度"  

### 管理进度  
- 从右侧列表选择进度  
- 删除或添加子进度  
- 查看统计信息和树状图

2. 将模组 JAR 文件放入 `mods` 文件夹

## 🎯 进度类型详解  

### 击杀进度  
- **目标ID示例**：`minecraft:zombie`、`minecraft:skeleton`  
- **触发条件**：击杀对应生物  
- **适用场景**：怪物猎人挑战、BOSS击杀记录  

### 获得进度  
- **目标ID示例**：`minecraft:diamond`、`minecraft:netherite_ingot`  
- **触发条件**：拾取对应物品  
- **适用场景**：资源收集、稀有物品获取  

### 探索进度  
- **目标ID示例**：`minecraft:overworld`、`minecraft:the_nether`  
- **触发条件**：进入对应维度  
- **适用场景**：维度探索、世界旅行  

### 建筑进度  
- **目标ID示例**：`minecraft:dirt`、`minecraft:stone`  
- **触发条件**：放置对应方块  
- **适用场景**：建筑工程、装饰统计  

### 附魔进度  
- **目标ID示例**：`minecraft:sharpness`、`minecraft:protection`  
- **触发条件**：获得对应附魔  
- **适用场景**：附魔收集、装备强化  

## 🖥️ 界面操作指南  

### 主界面区域  
- **左侧**：进度创建和设置区域  
- **右侧**：进度列表和统计显示区域  
- **底部**：功能按钮区域  

### 主题切换  
支持 6 种界面主题：  
- **默认**：经典暗色主题  
- **暗色**：更深沉的暗色系  
- **亮色**：明亮的浅色主题  
- **绿色**：自然绿色风格  
- **蓝色**：科技蓝色风格  
- **紫色**：神秘紫色风格  

### 子进度系统  
- 为现有进度添加子目标  
- 形成层次化的进度结构  
- 树状图显示完整进度关系  

## 🔍 常见问题  

### Q: 如何获取正确的目标 ID？  
A: 按 F3+H 打开高级提示框，将鼠标指向物品或生物即可查看 ID。  

### Q: 进度数据保存在哪里？  
A: 数据保存在世界文件夹下的 `simpleprogress` 目录中。  

### Q: 支持多人游戏吗？  
A: 是的，每个玩家有独立的进度数据。  

### Q: 进度会随游戏重启丢失吗？  
A: 不会，进度数据会自动保存和加载。
